## Governance rules

An agent must self-check generated code against these before presenting it.

- **no-raw-hex** [error] — Do not hardcode color literals (hex, oklch(), rgb()) in app or component code. Colors are authored once in src/styles/palette.css (and tokens/*.json) and consumed via tokens.
  - Fix: Use a semantic token: a palette.css var (e.g. var(--surface-1), var(--accent-danger)), a tokenized Tailwind utility (bg-surface-*, text-slate-*, bg-state-*), or an @/primitives/accentHex export (ACCENT_HEX, SLATE_HEX, MARKER_HEX) where a literal string is required (Cesium, SVG, Canvas).
- **no-arbitrary-type-and-color-classes** [error] — Do not use arbitrary Tailwind font sizes (text-[Npx]) or hex colors (-[#hex]). The type scale and color ramps are declared in src/styles/tailwind.css.
  - Fix: Type: text-3xs / text-2xs / text-xs-plus / text-xs / text-sm-minus / text-sm. Color: bg-surface-*, text-slate-*, bg-accent-*-soft, bg-state-*.
- **accenthex-generated** [error] — accentHex.ts is a generated mirror of palette.css's oklch tokens — never hand-edit a hex value there without changing the palette first.
  - Fix: Change the oklch value in src/styles/palette.css, then update the mirrored hex (the CI check converts oklch to hex and fails on drift).
- **icons-central-only** [error] — Import glyphs from @/lib/icons/central, never from lucide-react or @central-icons-react/* directly. The central wrapper is the only module allowed to know which icon library ships.
  - Fix: import { IconName } from '@/lib/icons/central'. If the glyph is missing, add it to the wrapper (prefer a filled Central glyph).
- **prefer-filled-icons** [warn] — Prefer the filled icon variant over the outlined/line variant whenever a filled twin exists.
  - Fix: Use the *Filled export from @/lib/icons/central (e.g. BellFilled); add the filled export to central.ts if it is missing.
- **interactive-radius** [warn] — Interactive elements (buttons, badges, chips) use the 4px 'rounded' radius — the system's signature corner.
  - Fix: Apply rounded (4px) / var(--c2-radius-interactive); avoid ad-hoc rounded-md|lg|full on interactive controls.
- **rtl-containers** [warn] — User-facing containers are RTL. Use logical Tailwind utilities (ms-*, me-*, ps-*, pe-*, start-*, end-*) for spacing; reserve rtl:/ltr: for physical outcomes like icon mirroring.
  - Fix: Set dir="rtl" on user-facing roots and replace left/right utilities with their logical equivalents.
- **oklch-status-colors** [info] — Danger and warning action colors are authored in oklch so hover/active stay perceptually even.
  - Fix: Author destructive/warning ramps as oklch in tokens/core.json (action.danger / action.warning).
- **card-slots-via-hook** [warn] — Build TargetCard content with the useCardSlots hook from a Detection object; never hardcode card slot data.
  - Fix: const slots = useCardSlots(detection, callbacks, context); pass slots into TargetCard.
- **tokens-single-source** [info] — Colors and the interactive radius are defined only in tokens/*.json (generated into tokens.generated.ts + tokens.generated.css). Do not introduce new color scales elsewhere.
  - Fix: Add the value to tokens/core.json (or tokens/c2-domain.json for tactical accents) and run `pnpm tokens:build`.
