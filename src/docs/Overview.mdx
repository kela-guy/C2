# CUAS Component Library

CUAS Target Management Card System – Component Architecture & Developer Reference

## System Overview

The CUAS (Counter-Unmanned Aircraft System) UI is built around a card-based architecture
for managing target detections, tracking, and mitigation. The system supports 5 operational
flows covering the full lifecycle from initial detection through neutralization and BDA
(Battle Damage Assessment).

## Card Anatomy

The card system is composed of two main layers:

- **SystemCard** — The outer header containing status chip, timer, classification badge, and chevron toggle
- **ExpandedTargetDetails** — The inner expandable content area with telemetry, actions, sensor data, and mission controls

## Primitives

| Component | Description |
|-----------|-------------|
| `StatusChip` | Colored label pill for detection status |
| `ActionButton` | Multi-variant action button with tooltip |
| `AccordionSection` | Collapsible section with framer-motion |
| `TelemetryRow` | RTL label/value row for telemetry data |
| `CollapsibleGroup` | Expandable group with count badge |
| `MissionPhaseChip` | Mission phase indicator with pulse |
| `MissionTimeline` | Step-by-step mission progress tracker |

## Operational Flows

| Flow | Description |
|------|-------------|
| **Flow 1** | Suspicion → Investigation → Decide → Act → Closure |
| **Flow 2** | Detection → Classification → Mitigation → BDA → Closure |
| **Flow 3** | Drone Deployment → Flying → On Station → RTB → Landed |
| **Flow 4** | Mission Planning → Execution → Confirmation → Complete |
| **Flow 5** | Post-Mitigation → BDA → Closure |
